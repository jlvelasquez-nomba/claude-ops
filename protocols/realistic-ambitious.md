## Outline (to expand when this protocol gets written)

WHEN: facing scope or ambition decision.
THEN: ask: what's the MOST AMBITIOUS thing we can ship in current constraints? not max ambition (overcommits), not min ambition (boring). the optimum is realistic-ambitious.
NEVER: settle for trivial just because ambitious is hard. NEVER overcommit to sky-high just because the user wants it.

## Example slot

Path B scoping 2026-05-22: proposed Path A (zod schema only, 1 day) vs Path B (full bridge, 1-2 weeks). Right call = Path A as foundation, B as follow-up. Realistic-ambitious.
