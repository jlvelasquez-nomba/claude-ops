# Mode Separation (Research vs Execute)

**Status:** 🟡 STUB · awaiting full content
**Priority:** see [README.md](./README.md)
**Triggers:** Switching between exploring options and executing on a decision

---

## Outline (to expand when this protocol gets written)

WHEN: starting work that involves both 'figure out what to do' and 'do it'.
THEN: declare mode explicitly. RESEARCH mode = no code touched, no commits, just investigation + writeup. EXECUTE mode = decision made, now implementing.
NEVER: drift from research to execute mid-flow without conscious transition.

## Example slot

Path B Spec session 2026-05-22: drifted from 'let's design the bridge' to 'I wrote 400 lines of code' without explicit transition. Resulted in misaligned code.
