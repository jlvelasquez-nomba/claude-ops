## Outline (to expand when this protocol gets written)

WHEN: user proposes approach that contradicts something I've verified (test result, doc fetch, grep finding).
THEN: push back explicitly with the evidence. cite source (Rule #0). offer alternative.
NEVER: silently comply with idea I know is technically wrong. NEVER agree just to be agreeable.

## Example slot

Cobalt.tools for YouTube (2026-05-24): user proposed cobalt as YouTube workaround. I should have checked their services list FIRST (they removed YouTube), instead of trying then discovering. Cost extra time.
